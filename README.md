# 物流員工管理系統

依據 `logistics_system_prompt.md` 規格實作的物流員工日常作業管理系統，包含首頁公告與行事曆、每日送件記錄、車輛里程記錄、今日角色（貨車司機/隨車人員）、請假申請與審核、薪資計算（含薪資單 PDF 匯出）、貨運行 Excel 對帳、車輛管理與保養提醒、員工帳號與歷史紀錄管理、管理者儀表板。

詳細開發歷程與各功能決策請見 `討論紀錄_2026_06_14.md`。

## 技術架構

- **後端**：Node.js + Express + TypeScript + Prisma + PostgreSQL
- **前端**：React + Vite + TypeScript + Tailwind CSS + React Router
- **認證**：JWT（第一個註冊的帳號會自動成為管理者 ADMIN；後台「開放員工自行註冊」開關關閉時，`/register` 僅允許建立第一個帳號）
- **角色**：
  - **ADMIN（管理者）**：完整權限。
  - **MANAGER（主管）**：填寫自己的每日送件記錄／車輛里程／今日角色；可查看員工列表、薪資查詢、派遣紀錄、貨運行對帳、管理者儀表板；可編輯任一員工的今日角色、車輛保養項目與維修登記、行事曆活動、首頁公告，並可審核／拒絕／刪除請假申請。員工帳號與角色管理、送件/里程紀錄編輯刪除、薪資扣款、對帳上傳、後台基礎設定、員工歷史紀錄清理僅 ADMIN 可操作。
  - **EMPLOYEE（員工）**：僅能存取自己的每日送件記錄、車輛里程、薪資、請假申請。
- **部署**：Railway（push 到 GitHub `main` 分支後自動重新部署並執行 `prisma migrate deploy`），本機開發以 `npx prisma generate` / `npx tsc --noEmit` / `npm run build` 做語法檢查為主

## 專案結構

```
backend/   Express API（路由、Prisma schema、薪資/對帳/車輛等業務邏輯）
frontend/  React 前端（員工/管理者頁面、路由、API client）
```

## 啟動方式

正式環境部署於 Railway（PostgreSQL），本機開發環境無法連接該資料庫，因此本機僅用於語法/型別檢查與前端介面開發：

### 1. 後端 API（語法檢查／編譯）

```powershell
cd backend
npm install
npm run prisma:generate
npx tsc --noEmit
npm run build
```

環境變數設定於 `backend/.env`（可參考 `.env.example`）：

```
DATABASE_URL="postgresql://..."
JWT_SECRET="change-this-secret-in-production"
JWT_EXPIRES_IN="7d"
PORT=4000
```

### 2. 前端網站（預設 http://localhost:5173）

```powershell
cd frontend
npm install
npm run dev
```

前端開發伺服器已設定 `/api` proxy 轉發至後端的 `http://localhost:4000`，因此開發時兩個伺服器需同時啟動（後端需連到可用的 PostgreSQL）。

## 帳號

- 第一個透過「註冊」建立的帳號會自動成為管理者（ADMIN）。
- 後台「開放員工自行註冊」開關關閉後，`/register` 僅允許建立第一個帳號，其餘員工帳號由管理者於「員工管理」建立。
- 正式環境（Railway）已有兩組測試帳號，可直接登入體驗各模組：
  - 管理者：`test-admin@example.com` / `test1234`
  - 一般員工：`test-employee@example.com` / `test1234`

## 已實作的模組

1. **首頁（公告欄＋行事曆）**：單則公告（ADMIN/MANAGER 可編輯，EMPLOYEE 唯讀）；當月行事曆顯示已核准請假與公司活動（ADMIN/MANAGER 可新增/刪除活動）。
2. **每日送件記錄**：正/逆物流件數、備註、今日角色（無/貨車司機/貨車隨車人員），每人每日一筆（重新填寫即覆蓋）；管理者可在薪資計算頁的每日明細中新增、編輯或刪除任一員工任一天的送件記錄（會同步刪除當日角色登記）。
3. **車輛里程記錄**：機車（必填）/貨車（選填）下拉選單、起訖里程、自動計算距離、自動帶入操作人員；管理者可在派遣紀錄頁編輯或刪除任一筆里程紀錄。
4. **請假申請與審核**：員工可申請/查看/取消請假，ADMIN/MANAGER 於「請假管理」頁核准、拒絕或刪除任何請假申請。
5. **薪資計算**：依出勤天數與日平均件數自動判定職稱（資深員工/員工/臨時工 × 高/低），管理者可逐月手動覆蓋；含職務加給、貨車司機/隨車加給、激勵獎金（出勤≥25天且日均件數>60→+3000，>30→+1500）、扣款項目；單一員工薪資單匯出 PDF（內嵌 Logo 與思源黑體，含每日明細與薪資計算說明），全公司薪資總表匯出 Excel。
6. **派遣紀錄**：依每日角色與里程紀錄即時統計「當天誰開哪輛車、誰是貨車司機/隨車人員」，供 ADMIN/MANAGER 查看與校正。
7. **貨運行 Excel 對帳**：上傳月結 Excel（讀取第一個工作表/總表，以關鍵字定位數值，避免版面變動造成抓錯），分別顯示「件數比對」（正/逆物流：貨運行 vs 系統 vs 差異）與「金額比對」（運費收入未稅：貨運行 vs 系統計算 vs 差異）。
8. **車輛管理**：機車/貨車分類、新建車輛自動建立預設保養項目、保養到期提醒（已逾期/即將到期）、車輛使用歷史紀錄、登記維修（ADMIN/MANAGER 皆可，會同步更新保養項目與累計里程）。
9. **員工管理**：帳號角色（ADMIN/MANAGER/EMPLOYEE）與特殊職稱（執行長/特殊）設定、啟用/停用、重設密碼、刪除帳號（僅 ADMIN，且無歷史紀錄才可直接刪除）。
10. **員工紀錄管理**（僅 ADMIN）：可查看單一員工所有歷史紀錄（送件、里程、今日角色、請假、扣款、職稱覆蓋），逐筆刪除或一鍵清空（需輸入員工姓名確認），供刪除帳號前清空資料使用。
11. **後台基礎設定**（僅 ADMIN）：薪資加給/扣款設定、每月正/逆物流稅前單價設定、註冊開關。
12. **管理者儀表板**：可選擇查看任一年月，顯示當月累計件數、預估薪資總支出、預估總收入、預估毛利等月結算統計；查看當月時額外顯示今日件數、車輛狀況、今日里程、待處理事項提醒，並提供快速連結。另有獨立的日期選擇器，可查看任一天所有在職員工的送件狀況（正/逆物流件數、今日角色、是否已填寫）。
13. **側邊導覽列**：依角色顯示對應選單，並依「核心作業／物流與派遣／人事行政／系統設定」等部門分類分區呈現（ADMIN/MANAGER 採同一套分類架構，EMPLOYEE 為簡化版本）。

## 開發狀態 / 後續可優化方向

- 詳細功能演進與決策記錄於 `討論紀錄_2026_06_14.md`，目前所有已討論需求皆已完成並部署。
- 尚未撰寫自動化測試（unit/e2e）。
- 薪資、對帳等規則若需依實際營運微調（例如職稱判定門檻、單價），可調整 `backend/src/services` 內對應檔案，或於後台「系統設定」調整單價。
