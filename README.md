# 物流員工管理系統

依據 `logistics_system_prompt.md` 規格實作的物流員工日常作業管理系統，涵蓋首頁公告與行事曆、每日送件記錄、車輛里程記錄、今日角色、請假申請與審核、薪資計算（含薪資單 PDF）、貨運行 Excel 對帳、車輛管理與保養提醒、員工帳號管理、管理者儀表板、區域管理、排班系統、加油回報與審核。

詳細開發歷程與各功能決策請見 `討論紀錄_2026_06_14.md`。

## 技術架構

- **後端**：Node.js + Express + TypeScript + Prisma + PostgreSQL
- **前端**：React + Vite + TypeScript + Tailwind CSS + React Router
- **認證**：JWT（第一個註冊的帳號自動成為董事長 ADMIN；後台「開放員工自行註冊」開關關閉時，`/register` 僅允許建立第一個帳號）
- **角色**：
  - **ADMIN（董事長）**：完整權限。
  - **MANAGER（執行長）**：填寫自己的每日送件記錄、車輛里程、今日角色；可查看員工列表、薪資查詢、派遣紀錄、貨運行對帳、管理者儀表板、區域管理；可編輯任一員工的今日角色、車輛保養項目與維修登記、行事曆活動、首頁公告，並可審核／拒絕／刪除請假申請、排班管理、油資審核。員工帳號與角色管理、送件/里程紀錄編輯刪除、薪資扣款、對帳上傳、後台基礎設定、員工歷史紀錄清理僅 ADMIN 可操作。
  - **REGION_MANAGER（區經理）**：填寫自己的每日送件記錄、車輛里程、今日角色（與 EMPLOYEE 相同）；另可透過「我的區域」頁面查看與管理自己所屬區域成員的送件狀況、薪資、派遣紀錄，並審核成員請假申請、排班、油資回報。無法存取全公司儀表板、薪資總表、貨運行對帳、員工帳號管理等全域功能。
  - **EMPLOYEE（員工）**：僅能存取自己的每日送件記錄、車輛里程、薪資、請假申請、排班查詢、加油回報。
- **部署**：Railway（push 到 GitHub `main` 分支後自動重新部署並執行 `prisma migrate deploy`），本機開發以 `npx prisma generate` / `npx tsc --noEmit` / `npm run build` 做語法檢查為主

## 專案結構

```
backend/   Express API（路由、Prisma schema、薪資/對帳/車輛等業務邏輯）
frontend/  React 前端（員工/管理者頁面、路由、API client）
```

## 啟動方式

正式環境部署於 Railway（PostgreSQL），本機開發環境無法連接該資料庫，因此本機僅用於語法/型別檢查與前端介面開發：

### 1. 後端 API（語法檢查／編譯）

```powershell
cd backend
npm install
npm run prisma:generate
npx tsc --noEmit
npm run build
```

環境變數設定於 `backend/.env`（可參考 `.env.example`）：

```
DATABASE_URL="postgresql://..."
JWT_SECRET="change-this-secret-in-production"
JWT_EXPIRES_IN="7d"
PORT=4000
```

### 2. 前端網站（預設 http://localhost:5173）

```powershell
cd frontend
npm install
npm run dev
```

前端開發伺服器已設定 `/api` proxy 轉發至後端的 `http://localhost:4000`，因此開發時兩個伺服器需同時啟動（後端需連到可用的 PostgreSQL）。

## 帳號

- 第一個透過「註冊」建立的帳號會自動成為董事長（ADMIN）。
- 後台「開放員工自行註冊」開關關閉後，`/register` 僅允許建立第一個帳號，其餘員工帳號由董事長於「員工管理」建立。
- 正式環境（Railway）已有兩組測試帳號，可直接登入體驗各模組：
  - 董事長：`test-admin@example.com` / `test1234`
  - 一般員工：`test-employee@example.com` / `test1234`

## 已實作的模組

1. **首頁（公告欄＋行事曆）**：單則公告（ADMIN/MANAGER 可編輯，EMPLOYEE 唯讀）；當月行事曆整合公司活動、已核准請假、排班資訊（綠色 badge，最多顯示 2 筆，超過以「+N 人」收合），任何人點擊日期可查看當日完整排班、請假、活動明細（ADMIN/MANAGER 可新增/刪除活動）。EMPLOYEE 登入後額外顯示「我的排班」快速欄，高亮今日排班區域並列出本月近期 4 筆。
2. **每日送件記錄**：正/逆物流件數、備註、今日角色（無/貨車司機/貨車隨車人員），每人每日一筆（重新填寫即覆蓋）；管理者可在薪資計算頁的每日明細中新增、編輯或刪除任一員工任一天的送件記錄（會同步刪除當日角色登記）。
3. **車輛里程記錄**：機車（必填）/貨車（選填）下拉選單，每天僅需填寫「今日結束里程」（累計里程），系統自動以該車輛前一筆紀錄計算當日行駛里程；管理者可在派遣紀錄頁編輯或刪除任一筆里程紀錄。
4. **請假申請與審核**：員工可申請/查看/取消請假，ADMIN/MANAGER 於「請假管理」頁核准、拒絕或刪除任何請假申請。
5. **薪資計算**：依出勤天數與日平均件數自動判定職稱（資深員工/員工/臨時工 × 高/低），管理者可逐月手動覆蓋；含職務加給、貨車司機/隨車加給、激勵獎金（出勤≥25天且日均件數>60→+3000，>30→+1500）、油資補貼（已核准加油回報加總）、扣款項目；單一員工薪資單匯出 PDF（內嵌 Logo 與思源黑體，含每日明細與薪資計算說明），全公司薪資總表匯出 Excel。
6. **派遣紀錄**：依每日角色與里程紀錄即時統計「當天誰開哪輛車、誰是貨車司機/隨車人員」；ADMIN/MANAGER 可校正任一在職員工當天的今日角色，即使該員工當天沒有車輛使用紀錄。
7. **貨運行 Excel 對帳**：上傳月結 Excel（讀取第一個工作表/總表，以關鍵字定位數值），分別顯示「件數比對」與「金額比對」，差異依正負以紅字／綠字標示。
8. **車輛管理**：機車/貨車分類、新建車輛自動建立預設保養項目、保養到期提醒、車輛使用歷史紀錄、登記維修（ADMIN/MANAGER 皆可，會同步更新保養項目與累計里程）。
9. **員工管理**：帳號角色（ADMIN/MANAGER/REGION_MANAGER/EMPLOYEE）與特殊職稱設定、啟用/停用、重設密碼、刪除帳號（僅 ADMIN，且無歷史紀錄才可直接刪除）。
10. **員工紀錄管理**（僅 ADMIN）：可查看單一員工所有歷史紀錄，逐筆刪除或一鍵清空（需輸入員工姓名確認），供刪除帳號前清空資料使用。
11. **後台基礎設定**（僅 ADMIN）：薪資加給/扣款設定、每月正/逆物流稅前單價設定、註冊開關、薪資計算公式設定（職稱判定門檻、每件單價、加給、激勵獎金條件）。
12. **管理者儀表板**：總覽頁（月結算統計、待處理事項、三個子頁面入口卡片）＋子頁面：
    - **每日營運總表**：依選定年月列出每一天的總正/逆物流、薪資、營收、盈餘、出勤人數、司機、跟車，並於表尾顯示平均與總計。
    - **員工送件狀況**：可選擇任一天，查看所有在職員工當天的送件狀況。
    - **車輛狀況**：顯示車輛目前累計里程、保養狀態與今日使用情形（僅於查看當月時提供）。
13. **側邊導覽列**：依角色顯示對應選單，依「核心作業／物流與派遣／人事行政／系統設定」等部門分類分區呈現。
14. **區域管理**（ADMIN/MANAGER，路由 `/regions`）：建立/編輯/停用區域，將在職員工指派至區域並設定區域經理；REGION_MANAGER 透過「我的區域」頁面（`/my-region`，分「今日送件狀況／成員列表／請假管理／派遣紀錄」四個分頁）查看與管理自己區域的成員。
15. **薪資計算公式設定**（僅 ADMIN，於「系統設定」頁）：可調整職稱判定門檻、各職稱每件單價、貨車司機/隨車人員加給、激勵獎金條件與金額，並填寫公式說明文字（顯示於薪資計算頁與我的薪資頁）。
16. **排班系統**（路由 `/schedule`，ADMIN/MANAGER/REGION_MANAGER 管理；`/my-schedule` 員工查看）：月曆視圖 + 列表視圖切換；點擊日期彈出 Modal 可查看、新增（單人或批次多選）、編輯、刪除排班；小區域輸入有歷史記錄自動補全；員工可於「我的排班」頁查看自己的月排班與統計；排班資訊整合至首頁行事曆（所有人可見）。
17. **加油回報與審核**（路由 `/fuel-report` 員工提交；`/fuel-review` 管理審核）：
    - 員工提交加油回報（日期、金額、車輛類型機車/貨車切換後選擇對應車牌、備註）；待審核中可撤回。
    - 管理者審核頁分三個 tab：**待審核**（核准/駁回操作）、**歷史紀錄**（依月份/員工篩選）、**車輛油費**（依月份分別統計機車／貨車已核准油費合計，各自可展開明細）。
    - 已核准的加油回報自動計入當月薪資（油資補貼），並顯示於員工「我的薪資」頁與薪資單 PDF。

## 開發狀態 / 後續可優化方向

- 詳細功能演進與決策記錄於 `討論紀錄_2026_06_14.md`，目前所有已討論需求皆已完成並部署。
- 尚未撰寫自動化測試（unit/e2e）。
- 薪資、對帳等規則若需依實際營運微調（例如職稱判定門檻、單價），可於後台「系統設定」調整，或修改 `backend/src/services` 內對應檔案。
