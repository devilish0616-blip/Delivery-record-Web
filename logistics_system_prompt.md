# 物流員工管理系統 — 開發需求 Prompt

## 專案概述

請幫我設計並開發一套「物流員工日常作業管理系統」，用於記錄員工每日送件狀況、車輛使用里程，並提供後台薪資計算與貨運行對帳功能。系統以**網頁為主（RWD響應式設計，支援手機瀏覽器操作）**，不需要原生 App。

---

## 使用者角色

| 角色 | 說明 |
|------|------|
| **員工** | 每日登入後，輸入當日送件資料與車輛里程 |
| **管理者（Admin）** | 擁有完整後台權限，可查看所有員工資料、計算薪資、處理對帳 |

系統需要帳號註冊／登入機制，管理者由開發者直接指定（或透過後台升級權限），員工透過邀請或自行註冊。

---

## 功能模組

### 模組一：每日送件記錄（員工操作）

員工每天登入後，輸入當日作業資料：

- **正物流件數**：當日送出件數（手動輸入數字）
- **逆物流件數**：當日收回件數（手動輸入數字）
- **備註**：選填，記錄異常或說明
- **日期**：預設當天，可修改補登

資料送出後，員工可查看自己的歷史紀錄（按日期列表）。

---

### 模組二：車輛里程記錄（員工操作）

每日完成送件後記錄車輛使用狀況：

- **車牌號碼**：下拉選單選擇（車輛清單由管理者在後台維護，見模組五）
- **當日起始里程**：手動輸入（公里）
- **當日結束里程**：手動輸入（公里）
- **當日行駛里程**：自動計算（結束 - 起始）
- **使用人員**：自動帶入登入員工姓名
- **日期**：預設當天

---

### 模組三：薪資計算（後台管理者操作）

#### A. 按件計酬員工 — 職稱自動判定邏輯

每月結算時，系統根據員工**當月出勤天數**與**日平均件數（正＋逆合計）**，自動判定職稱與適用單價。

> 特殊職稱（執行長、特殊）由管理者手動在員工設定中指派，不參與自動判定。

##### Step 1：依出勤天數判定職稱大類

| 當月出勤天數 | 職稱大類 |
|---|---|
| >= 20 天 | 資深員工 |
| > 10 天（且 < 20 天） | 員工 |
| <= 10 天 | 臨時工 |

##### Step 2：依日平均件數判定高/低（僅資深員工、員工適用）

- 日平均件數 = 當月（正物流 + 逆物流）合計 ÷ 出勤天數
- 日平均 > 60 件 → **高**
- 日平均 <= 60 件 → **低**

##### Step 3：每日實際計薪單價

每日薪資根據**當日件數**決定單價，全日同一單價（非分段計算）：

| 職稱 | 當日件數條件 | 適用單價 |
|---|---|---|
| 執行長 | 任何件數 | **30 元**（固定，不分件數） |
| 資深員工（高） | 當日 <= 100 件 | 25 元 |
| 資深員工（高） | 當日 > 100 件 | 28 元（全部件數適用） |
| 資深員工（低） | 當日 <= 100 件 | 23 元 |
| 資深員工（低） | 當日 > 100 件 | 26 元（全部件數適用） |
| 員工（高） | 當日 <= 100 件 | 25 元 |
| 員工（高） | 當日 > 100 件 | 28 元（全部件數適用） |
| 員工（低） | 當日 <= 100 件 | 23 元 |
| 員工（低） | 當日 > 100 件 | 26 元（全部件數適用） |
| 臨時工 | 任何件數 | **23 元**（固定） |
| 特殊 | 任何件數 | **30 元**（固定） |

> **當日 > 100 件時，當日所有件數全部套用高單價（非分段，非只算超過的部分）**

##### Step 4：月薪計算公式

```
當月薪資 = Σ（每日件數 × 當日適用單價）
```

- 月初先以「預估職稱」試算，月底出勤確定後重新以「最終職稱」結算
- 系統顯示每日明細（日期、件數、單價、小計）與月薪總計

---

#### B. 司機與隨車人員（日薪加給）

- 每日派遣紀錄：記錄當日哪位員工擔任**司機**、哪位擔任**隨車人員**
- 司機：每出勤一天 +1,000 元
- 隨車人員：每出勤一天 +500 元
- 月底加總出勤天數自動計算加給

> 同一員工可能同時有按件薪資 + 司機/隨車加給，系統合併計算並分項顯示。

---

管理者後台可：
- 查看每位員工當月薪資明細（含職稱判定依據、每日件數與單價）
- 手動調整特定員工的職稱（覆蓋自動判定結果）
- 匯出薪資報表（Excel）

---

### 模組四：貨運行 Excel 對帳與收款計算（後台管理者操作）

每月貨運行會提供一份固定格式的 Excel 檔案，記錄當月我方透過貨運行送出的總件數與金額。

功能需求：
1. **上傳 Excel 檔案**：管理者上傳貨運行提供的月結 Excel
2. **解析資料**：系統自動讀取關鍵欄位（送件數量、金額等）
3. **對帳比對**：與系統內員工自行記錄的送件總數進行比對，標示差異
4. **計算實收金額**：
   - 貨運行應付金額 × (1 - 9%) = 實際入帳金額
   - （靠行抽成 9%，系統自動計算）
5. **顯示對帳報告**：含原始金額、抽成金額、實收金額、件數差異

---

### 模組五：後台基礎設定（管理者操作）

管理者可在後台自由維護系統所需的基礎資料，不需要修改程式碼：

#### 車輛管理
- **新增車輛**：輸入車牌號碼、車型／備註（選填）、啟用狀態
- **編輯車輛**：修改車牌或備註
- **停用車輛**：停用後不再出現於員工的下拉選單，但歷史紀錄保留
- **車輛列表**：顯示所有車輛及其當月累計里程

#### 換機油提醒
- 管理者可為每台車輛記錄**上次換機油時的里程數**
- 系統持續追蹤每台車的當前累計里程
- 當（當前里程 - 上次換機油里程）>= 1,000 km 時，後台自動顯示警示提醒
- 換完機油後，管理者點擊「已換機油」更新基準里程，提醒重置
- 儀表板顯示各車輛距下次換機油的剩餘里程，快到時（如剩餘 < 200 km）以顏色標示警示

#### 員工管理
- 查看所有員工帳號
- 設定員工角色（員工 / 管理者）
- 指派特殊職稱：「執行長」或「特殊」（這兩種不參與自動判定，固定單價）
- 一般員工職稱每月由系統自動判定，管理者可手動覆蓋
- 停用員工帳號

#### 薪資參數設定
- 司機日薪加給金額（預設 +1,000，可調整）
- 隨車人員日薪加給金額（預設 +500，可調整）

#### 每月收入單價設定（每月填寫一次）

管理者每月設定公司向貨運行收取的每件單價（稅前），系統自動計算稅後實收：

| 欄位 | 說明 |
|---|---|
| 正物流每件稅前單價 | 管理者手動填入（元） |
| 正物流每件稅後單價 | 系統自動計算：稅前 × 0.91 + 稅前 × 0.05 = 稅前 × **0.96** |
| 逆物流每件稅前單價 | 管理者手動填入（元） |
| 逆物流每件稅後單價 | 系統自動計算：稅前 × 0.96 |

> 此單價為**公司收入面**，與員工薪資的計薪單價（25/28/30元等）為兩個獨立欄位，意義不同。

系統根據此設定，每月自動計算：

```
正物流總收入   = 當月正物流總件數 × 正物流稅後單價
逆物流總收入   = 當月逆物流總件數 × 逆物流稅後單價
當月總收入     = 正物流總收入 + 逆物流總收入
當月總人力成本  = Σ 所有員工薪資（含按件 + 司機/隨車加給）
當月毛利       = 當月總收入 - 當月總人力成本
```

管理者後台可查看月度收支摘要，並可與貨運行 Excel 對帳結果互相核對。

## 後台管理者儀表板

管理者登入後應看到：

- 今日全員送件總件數（正 / 逆物流分開顯示）
- 本月累計件數與預估薪資總支出
- 本月預估總收入（正＋逆物流件數 × 稅後單價）
- 本月預估毛利（總收入 - 總人力成本）
- 車輛今日使用狀況
- 待處理事項提示（例如：尚未對帳的月份、尚未設定本月單價、車輛需要換機油）
- 快速進入各功能模組的入口

## 技術建議

請以最適合此需求的技術棧進行開發，建議如下：

| 層級 | 建議技術 |
|------|----------|
| 前端 | React（或 Vue 3）+ Tailwind CSS，RWD 設計 |
| 後端 | Node.js（Express）或 Python（FastAPI） |
| 資料庫 | PostgreSQL 或 MySQL |
| 認證 | JWT Token，角色權限控管（admin / employee） |
| Excel 處理 | SheetJS（前端解析）或 openpyxl（後端解析） |
| 部署 | Docker 容器化，建議可部署至 VPS 或雲端 |

---

## 非功能需求

- **RWD**：員工在手機瀏覽器也能順暢輸入資料
- **資料安全**：員工只能看到自己的資料，無法查看其他員工
- **操作簡單**：員工端介面盡量精簡，減少學習成本
- **繁體中文介面**

---

## 開發優先順序（建議）

1. 帳號系統（登入 / 角色管理）
2. 後台基礎設定（車輛管理、員工管理、薪資參數）
3. 每日送件記錄功能
4. 車輛里程記錄功能
5. 薪資計算後台
6. Excel 對帳功能
7. 儀表板與報表匯出

## 補充說明

- 貨運行 Excel 格式**固定**，每月欄位結構相同，可事先對應欄位 mapping
- 司機與隨車人員每天一組（一名司機 + 一名隨車），需有每日派遣紀錄表
- 管理者（開發者本人）為最高權限，不受任何限制
