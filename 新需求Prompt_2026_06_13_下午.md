## 新功能需求

### 功能 1：首頁公告欄 + 行事曆

#### 公告欄
- 所有角色登入後**第一個看到的頁面**改為「公告欄 + 行事曆」首頁
- 公告欄只有**一則公告**（最新的覆蓋舊的），純文字格式
- MANAGER 和 ADMIN 可以編輯公告內容，EMPLOYEE 只能查看
- 後端新增：
  - `GET /api/announcement`：取得目前公告
  - `PUT /api/announcement`：更新公告（僅 ADMIN/MANAGER）

#### 行事曆
- 首頁下方顯示當月行事曆
- 行事曆上顯示兩種資料：
  1. **請假紀錄**：已審核通過的請假（顯示員工姓名 + 請假）
  2. **公司活動／重要日期**：由 ADMIN/MANAGER 直接在行事曆上新增
- ADMIN/MANAGER 可以點擊行事曆上的日期新增活動（輸入活動名稱）、刪除活動
- EMPLOYEE 只能查看，不能新增／刪除
- 後端新增：
  - `GET /api/events`：取得所有活動
  - `POST /api/events`：新增活動（僅 ADMIN/MANAGER）
  - `DELETE /api/events/:id`：刪除活動（僅 ADMIN/MANAGER）

### 功能 2：員工請假申請與審核

#### 員工端（EMPLOYEE / MANAGER / ADMIN 都可申請）
- 導覽列新增「請假申請」頁面
- 員工可以：
  - 新增請假申請（選擇日期、填寫原因）
  - 查看自己的請假紀錄與審核狀態（待審核／已核准／已拒絕）
  - 取消尚未審核的請假申請

#### 管理者端（ADMIN / MANAGER 審核）
- 後台新增「請假管理」頁面
- 顯示所有員工的請假申請列表
- 可以**核准**或**拒絕**每筆申請
- 假別：只有一種「請假」，不區分假別
- 請假不影響薪資計算，只是記錄出勤狀況

#### 資料表
新增 `LeaveRequest`：
```
id
userId（申請人）
date（請假日期）
reason（原因，選填）
status（PENDING / APPROVED / REJECTED）
reviewedBy（審核人 userId，可為空）
reviewedAt（審核時間，可為空）
createdAt
```

#### 後端 API
- `POST /api/leaves`：新增請假申請（登入者皆可）
- `GET /api/leaves/my`：取得自己的請假紀錄
- `GET /api/leaves`：取得所有人請假紀錄（僅 ADMIN/MANAGER）
- `PATCH /api/leaves/:id/approve`：核准（僅 ADMIN/MANAGER）
- `PATCH /api/leaves/:id/reject`：拒絕（僅 ADMIN/MANAGER）
- `DELETE /api/leaves/:id`：取消申請（本人且狀態為 PENDING）
