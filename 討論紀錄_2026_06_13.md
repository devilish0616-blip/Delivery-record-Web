## 系統現況

- GitHub repo：`https://github.com/devilish0616-blip/Delivery-record-Web`
- 部署：Railway（push 到 GitHub `main` 分支後會自動重新部署，後端會自動執行 `prisma migrate deploy`）
- 技術架構：
  - 後端：Node.js + Express + TypeScript + Prisma + PostgreSQL（Railway 上）
  - 前端：React + Vite + TypeScript + Tailwind CSS + react-router-dom
- 角色：ADMIN（管理者）、MANAGER（主管）、EMPLOYEE（員工）
- ⚠️ 本機開發環境 `backend/.env` 仍是 SQLite，與 `schema.prisma`（PostgreSQL）不一致，
  本機無法直接啟動連資料庫的後端。日常開發以 **Railway 上的部署版本為主**，
  本機只用 `npx prisma generate` / `npx tsc --noEmit` / `npm run build` 做語法與型別檢查。

## 已完成功能（依完成時間排序）

### 1. 管理者編輯員工每日送件記錄（commit `3d5639e`）

- 新增管理者專用 API：`PUT /api/deliveries/:userId/:date`，可修正指定員工指定日期的正/逆物流件數與備註。
- 沿用 `upsert`（依 `userId + date` 為鍵），員工同一天重新填寫送件記錄為「覆蓋機制」，不會產生重複紀錄。
- 薪資計算頁面（`SalaryPage.tsx`）的每日明細表格新增管理者編輯功能（展開員工明細後可點「編輯」修改當日件數）。

### 2. 車輛管理：機車/貨車分類 + 保養提醒（commit `bd07860`）

- `Vehicle` 新增 `type` 欄位（`MOTORCYCLE` 機車 / `TRUCK` 貨車），新建車輛時依車型自動建立預設保養項目。
- 新增 `VehicleMaintenanceItem`（保養項目）資料表：項目名稱、更換週期（km）、上次更換里程/時間/備註。
- 預設保養項目：

  | 車型 | 保養項目 | 更換週期 |
  |---|---|---|
  | 機車 | 機油 | 每 1,000 km |
  | 機車 | 齒輪油 | 每 3,000 km |
  | 機車 | 空氣濾芯 | 每 5,000 km |
  | 機車 | 皮帶 | 每 12,000 km |
  | 貨車 | 機油 | 每 1,000 km |

- 車輛管理頁可展開查看保養項目清單；儀表板/車輛列表依「已逾期 / 即將到期」顯示提醒。
- 刪除車輛防呆：已有里程紀錄的車輛禁止刪除（需改為「停用」），刪除前需輸入完整車牌號碼確認。
- 編輯車輛新增「修正目前累計里程」欄位。

### 3. 大改版：主管角色、里程自填、今日角色、職務加給、激勵獎金、密碼重設、批次匯入（commit `2d966b8`）

這次是依照《需求清單 2026/06/13》一次完成以下 14 項，全部已完成並推送：

1. ✅ **Bug 修復**：管理者點「車輛管理」整頁空白的問題（加防呆 + 全域 ErrorBoundary）。
2. ✅ **新增 MANAGER（主管）角色**：可查看所有資料但原則上不能修改，例外是可編輯車輛保養項目、可查看派遣紀錄；主管也需要填寫每日送件記錄、車輛里程、今日角色。角色由 ADMIN 在員工管理頁指派。
3. ✅ **ADMIN 也能填寫每日送件記錄**：導覽列開放 ADMIN/MANAGER 進入「每日送件記錄」頁。
4. ✅ **車輛里程記錄改版**：員工自行記錄「今日車輛使用」，分為機車（必填）與貨車（選填）兩區塊，各自下拉選單依車型篩選啟用中車輛；以 `MileageRecord` 的 `@@unique([userId, date, vehicleId])` 做 upsert 覆蓋機制。
5. ✅ **新增「今日角色」**：員工可選擇「無 / 司機 / 隨車人員」，存入新的 `DailyRoleRecord`（`@@unique([userId, date])`）；ADMIN 可在後台修改任何人的今日角色。
6. ✅ **派遣紀錄頁改為唯讀統計頁**：移除原本手動派遣紀錄（`DispatchRecord` 資料表已 `DROP TABLE`，使用者已確認當時尚在測試、無正式資料，可接受刪除），改由 `MileageRecord` + `DailyRoleRecord` 即時統計「今天誰開哪輛車、誰是司機/隨車人員」。
7. ✅ **車輛管理新增「使用歷史」**：展開車輛時顯示最近使用紀錄（誰、哪天、起訖里程、當日角色），協助判斷是否停用。
8. ✅ **車輛類型可編輯**：編輯車輛時可修改車型（機車/貨車），變更前彈出確認對話框（「變更車型將會重置預設保養項目，確定要修改嗎？」），確認後重新套用對應車型的預設保養項目。
9. ✅ **修正薪資頁編輯路徑 bug**：`/delivery/...` 修正為 `/deliveries/...`，管理者可正常編輯他人每日件數。
10. ✅ **單一員工匯出薪資單**：薪資頁新增「匯出薪資單」，產生該員工當月 Excel（出勤天數、每日明細、職稱判定、各項加給、激勵獎金、實領薪資）。
11. ✅ **職務加給**：`User.monthlyAllowance`，後台可針對個別員工設定固定每月加給，自動納入薪資計算與薪資單。
12. ✅ **員工管理新增「重設密碼」**：`PUT /api/employees/:id/password`（僅 ADMIN），前端跳出 modal 輸入新密碼+確認密碼。
13. ✅ **批次匯入過往送貨紀錄**：`POST /api/deliveries/batch-import`（僅 ADMIN），前端解析 Excel 預覽後送出 JSON 陣列，依姓名/Email 比對員工、以 `(userId, date)` upsert，回傳成功/失敗筆數與原因；含範本下載。
14. ✅ **激勵獎金**：出勤天數 ≥ 25 且日均件數 > 60 → +3000；≥25 且 >30 → +1500（IF/ELSE，不重複加），計入 `totalSalary`，薪資頁與 Excel 匯出皆顯示。

### 4. 派遣紀錄管理者編輯/刪除、員工帳號刪除（commit `6a79168`）

- **派遣紀錄管理者校正**：在「派遣紀錄統計」頁的車輛使用表格中，管理者可對每筆里程紀錄「編輯」（修改起/終里程）或「刪除」，方便員工填錯時由後台修正。對應後端 `PUT /api/mileage/:id`、`DELETE /api/mileage/:id`（僅 ADMIN）。
- **員工帳號刪除**：員工管理頁新增「刪除帳號」按鈕（僅 ADMIN，且不能刪除自己）。後端 `DELETE /api/employees/:id` 會檢查該帳號是否有送件/里程/角色/扣款/職稱覆蓋等歷史紀錄，**只有完全沒有歷史紀錄的帳號才能直接刪除**，否則提示改用既有的「停用帳號」功能，避免破壞薪資/歷史資料。

### 5. 機車維修登記彈出視窗（commit `c9c7594`）

- 車輛管理頁每台車的操作欄新增「登記維修」按鈕，**僅 MANAGER（主管）與 ADMIN 可見**。
- 點擊後跳出視窗，可填寫：
  - **目前里程**（預設帶入該車目前累計里程，可修改）
  - **維修項目**（下拉選單，列出該車所有保養項目，例如機油、齒輪油等）
  - **備註（選填）**
- 確認送出後：後端 `PATCH /api/vehicles/:id/maintenance/:itemId`（`requireAdminOrManager`）會更新該保養項目的「上次更換里程／時間／備註」，若填入里程大於車輛目前累計里程，也會同步更新車輛的累計里程。
- 原本「標記已更換」的行內編輯方式已移除，改為此彈出視窗操作；ADMIN 仍可刪除保養項目。
