# 討論紀錄 2026/06/13

## 1. 管理者後台編輯員工每日送件記錄

- 新增管理者專用 API：`PUT /api/delivery/:userId/:date`，可修正指定員工指定日期的正/逆物流件數與備註。
- 沿用原有的 `upsert`（依 `userId + date` 為鍵），確認**員工同一天重新填寫送件記錄為「覆蓋機制」**，不會產生重複紀錄。
- 前端「薪資計算」頁面（`SalaryPage.tsx`）的每日明細表格新增管理者編輯功能：
  - 展開某員工的每日明細後，管理者可點擊「編輯」，直接修改當日正/逆物流件數並儲存。

> 對應 commit：`3d5639e` 新增管理者編輯員工每日送件記錄功能

## 2. 車輛管理：區分機車 / 貨車並設定保養提醒

- `Vehicle` 新增 `type` 欄位（`MOTORCYCLE` 機車 / `TRUCK` 貨車），新建車輛時依車型自動建立預設保養項目。
- 新增 `VehicleMaintenanceItem`（保養項目）資料表，每個項目包含：
  - 項目名稱（機油、齒輪油、空氣濾芯、皮帶...等，可自訂）
  - 更換週期（公里）
  - 上次更換里程、上次更換時間、上次更換備註（記錄「在幾公里時換了什麼」）
- 預設保養項目：
  | 車型 | 保養項目 | 更換週期 |
  | --- | --- | --- |
  | 機車 | 機油 | 每 1,000 km |
  | 機車 | 齒輪油 | 每 3,000 km |
  | 機車 | 空氣濾芯 | 每 5,000 km |
  | 機車 | 皮帶 | 每 12,000 km |
  | 貨車 | 機油 | 每 1,000 km（沿用原有設定） |
- 後台「車輛管理」頁面：
  - 新增/編輯車輛時可選擇車型。
  - 每輛車可展開查看保養項目清單，管理者可：
    - 標記「已更換」並填寫備註（自動以目前累計里程作為新的基準里程）。
    - 新增自訂保養項目、刪除保養項目。
  - 儀表板與車輛列表會依「已逾期 / 即將到期（剩餘里程 < 週期的 20%）」顯示提醒。

## 3. 車輛刪除防呆機制與里程修正

- **刪除車輛**：
  - 後台新增 `DELETE /api/vehicles/:id`，若該車輛已有里程紀錄或派遣紀錄則禁止刪除（避免破壞歷史資料，需改為「停用」）。
  - 前端點擊刪除後會跳出確認對話框，**需輸入該車輛完整車牌號碼**才能啟用「確認刪除」按鈕，避免誤刪。
- **目前里程修正**：
  - 編輯車輛時新增「修正目前里程」欄位，管理者可直接更正車輛目前累計里程（`currentMileage`）。

> 對應 commit：`bd07860` 車輛管理：區分機車/貨車並依車型設定保養提醒項目，新增刪除確認與里程修正
