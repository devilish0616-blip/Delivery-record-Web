# 需求13：管理者批次匯入過往送貨紀錄

## 背景
管理者需要能夠補登員工過往的送貨紀錄，支援透過 Excel 一次匯入多天多人的資料。

## Excel 格式規範

| 欄位 | 說明 | 必填 |
|------|------|------|
| 員工姓名 or Email | 用來對應系統內的員工帳號 | ✅ |
| 日期 | 格式：YYYY/MM/DD，任何日期皆可 | ✅ |
| 正物流件數 | 數字 | ✅ |
| 逆物流件數 | 數字 | ✅ |
| 備註 | 文字，可空白 | ❌ |

範例：
```
員工姓名    日期          正物流件數  逆物流件數  備註
蚯蚓       2026/06/01    50         10
陳彥旭     2026/06/01    45         8          補登
李泓玫     2026/05/15    60         5
```

## 功能說明

### 前端
- 位置：後台「每日送件記錄」管理頁面，新增「批次匯入」按鈕
- 點擊後跳出視窗：
  1. 提供 Excel 範本下載按鈕
  2. 上傳 Excel 檔案
  3. 預覽匯入資料（顯示解析出的筆數、員工、日期範圍）
  4. 確認匯入按鈕
- 匯入結果顯示：
  - 成功幾筆
  - 失敗幾筆（找不到員工、格式錯誤等）並列出原因

### 後端
- 新增 API：`POST /api/delivery/batch-import`
- 僅 ADMIN 可呼叫
- 邏輯：使用 upsert（員工ID + 日期為唯一鍵），已有紀錄則覆蓋，沒有則新增
- 員工比對方式：先用姓名比對，找不到再用 Email 比對，都找不到則標記為失敗
